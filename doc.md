## 项目描述
一个“幽灵文本阅读器”，用于上班摸鱼。  
核心特性：
- 窗口无边框、完全透明（包括标题栏区域），上面只显示不透明的文本。
- 默认状态下窗口几乎不可见（不透明度设为 1，肉眼无法察觉），且鼠标穿透（点击会穿透到后面的窗口）。
- 当鼠标移入窗口区域时，窗口立即变为正常不透明（不透明度 1.0），同时关闭鼠标穿透，用户可以滚动阅读文本。
- 当鼠标移出窗口区域时，窗口恢复几乎不可见和鼠标穿透状态。
- 支持拖入或打开 .txt 文本文件，显示在窗口中，字体、大小、颜色可配置（默认用浅灰色，模仿 IDE 代码颜色，便于伪装）。
- 支持基础阅读功能：鼠标滚轮滚动文本、键盘上下翻页、PgUp/PgDn、Home/End。
- 窗口始终置顶。
- 窗口尺寸可调，但默认大小应适合阅读（如 800x600 或类似），且可以记住上次关闭时的位置和大小（保存到本地配置）。
- 包含简单的老板键功能：按下特定的全局热键（如 Ctrl+Shift+H）可以彻底隐藏/显示阅读器窗口（在系统托盘或直接隐藏进程窗口）。

## 技术栈
- Python 3
- PyQt5（或 PySide6，但推荐 PyQt5，因为资料多）
- 不需要外部资源文件，界面用代码生成。

## 实现细节与要点

### 1. 窗口设置
- 使用 `QMainWindow` 或 `QWidget` 作为主窗口，设置窗口标志：
  - `Qt.FramelessWindowHint` (无边框)
  - `Qt.WindowStaysOnTopHint` (置顶)
  - `Qt.Tool` (可选，让窗口不在任务栏显示，但可能需要调整，如果不显示任务栏更方便摸鱼就用 Tool 标志)
- 窗口背景透明：`setAttribute(Qt.WA_TranslucentBackground)`，这样窗口背景完全透明。
- 文本区域使用 `QTextEdit` 或 `QPlainTextEdit`，设置为只读，背景透明，文字颜色可配，无边框。
- 整个窗口的不透明度控制通过 `setWindowOpacity()` 实现：极低值 0.01（即1%），而不是 0，因为设为 0 会导致鼠标检测失效。

### 2. 鼠标检测与状态切换（核心逻辑）
- 因为默认状态下窗口需要鼠标穿透，所以必须启用 `setAttribute(Qt.WA_TransparentForMouseEvents, True)` 让窗口不接收鼠标事件。
- 但这样一来，鼠标进入窗口时就无法触发 `enterEvent`。因此，我们不能使用 Qt 的鼠标事件来激活显示。
- 解决方案：使用一个 `QTimer`，每隔 50 毫秒轮询鼠标全局坐标 (`QCursor.pos()`)，并与窗口的几何矩形 (`self.geometry()`) 做包含判断。
  - 当鼠标在矩形内，并且当前窗口处于“隐藏”模式时：执行“显现”动作：`setWindowOpacity(1.0)`，同时关闭鼠标穿透 (`setAttribute(Qt.WA_TransparentForMouseEvents, False)`)，以便用户可以点击和滚动文本。
  - 当鼠标不在矩形内，并且当前窗口处于“显现”模式时：执行“隐藏”动作：`setWindowOpacity(0.01)`，开启鼠标穿透。
  - 添加一个状态变量防止重复设置。
- 注意：滚动条区域也属于窗口的一部分，鼠标在滚动条上时也应该显示窗口，所以几何检测直接用整个窗口矩形即可。

### 3. 文本加载与显示
- 支持通过菜单栏（虽然无边框，但可以自定义一个右上角按钮或右键菜单）或直接拖拽文件来打开 .txt 文件。
- 实现 `dragEnterEvent` 和 `dropEvent` 接受文本文件，读取内容用 `utf-8` 或 `gbk` 自动尝试。
- 文本显示在 `QTextEdit` 中，设置字体默认 `Consolas` 或 `Microsoft YaHei`，大小 12pt，颜色默认 `#CCCCCC`（亮灰，像代码），可加简单配置。
- 支持鼠标滚轮：因为默认状态下穿透，所以显示后需要确保滚轮事件传递给 `QTextEdit`。

### 4. 老板键（全局热键）
- 使用 PyQt 的 `QHotkey` 库可能不够稳定，推荐使用 `keyboard` 库进行全局监听，或者直接用 `pynput`。
- 但为了减少依赖，可以考虑用 Windows API 注册热键，例如通过 `win32gui` 或 `ctypes`。
- 简化方案：在程序中增加系统托盘图标（`QSystemTrayIcon`），并提供隐藏/显示菜单。同时监听全局热键 `Ctrl+Shift+H`，可使用 `keyboard` 库实现（需额外安装，但很简单）。
- 如果不想引入额外库，可以用 `QShortcut` 但只对程序窗口有效，不符合“全局”要求。因此请使用 `keyboard` 库，在单独线程监听，并在检测到热键时发射 Qt 信号以切换窗口可见性（注意线程安全，使用 `pyqtSignal`）。

### 5. 配置持久化
- 使用 `QSettings` 保存和加载以下信息：
  - 窗口位置 (x, y)
  - 窗口大小 (width, height)
  - 上次打开的文件路径（可选，程序启动时可以自动加载）
  - 字体、字号、文字颜色
- 在窗口移动或调整大小时更新这些设置（`moveEvent` 和 `resizeEvent` 中延迟保存）。

### 6. 窗口移动和调整大小
- 由于无边框，需要自己实现窗口拖动：重写 `mousePressEvent`、`mouseMoveEvent` 记录偏移，移动窗口。仅当鼠标在文本以外的区域（比如自定义标题栏区域）时才拖动，但本需求中没有标题栏，所以可以设定为：如果鼠标在窗口边缘的 10 像素内，则进行大小调整；否则点击文本区时不拖动窗口。更简单的做法：在窗口顶部放一个极窄（如 30px）的透明 `QWidget` 作为标题栏，鼠标在此区域可拖动窗口，这个标题栏在显现模式下才可见（同样背景透明）。请实现这个窄条，方便拖动窗口位置。
- 边缘拉伸：可以使用 `QSizeGrip` 放在右下角，或者手动处理 `nativeEvent` 实现边缘拖拽调整大小。建议在右下角放置一个半透明的 `QSizeGrip`，在显现模式下可见。

### 7. 右键菜单
- 在文本区域右键弹出菜单，包含：
  - 打开文件
  - 切换始终置顶
  - 设置字体/颜色（简单的 QFontDialog 和 QColorDialog）
  - 老板键说明
  - 退出程序
